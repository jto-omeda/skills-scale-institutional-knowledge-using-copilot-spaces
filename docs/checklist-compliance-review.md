# Compliance Review Checklist

## Purpose
This checklist helps Compliance Leads and project teams systematically review projects for regulatory, security, and policy compliance. Use this at key project milestones to identify and address compliance requirements early.

## Section 1: Data Privacy & Protection

### Personal Data Handling
- [ ] Identified what personal data (PII) the project collects, processes, or stores
- [ ] Documented lawful basis for processing personal data
- [ ] Implemented data minimization principles (collect only what's necessary)
- [ ] Defined data retention and deletion policies
- [ ] Ensured user consent mechanisms are in place (where required)
- [ ] Provided mechanisms for data subject rights (access, deletion, portability)
- [ ] Documented data flows and third-party data sharing
- [ ] Implemented appropriate data encryption (at rest and in transit)

## Section 2: Security Requirements

### Security Controls
- [ ] Conducted threat modeling for new features/systems
- [ ] Implemented authentication and authorization controls
- [ ] Applied principle of least privilege for access controls
- [ ] Enabled security logging and monitoring
- [ ] Implemented input validation and sanitization
- [ ] Protected against common vulnerabilities (OWASP Top 10)
- [ ] Configured secure defaults and hardened configurations
- [ ] Planned for security patch management

### Security Testing
- [ ] Completed security code reviews
- [ ] Ran automated security scanning (SAST/DAST)
- [ ] Performed dependency vulnerability scanning
- [ ] Conducted penetration testing (if required)
- [ ] Validated encryption implementations

**Notes/Gaps:**

---

## Section 3: Access Controls & Identity Management

- [ ] Defined role-based access control (RBAC) requirements
- [ ] Implemented multi-factor authentication (MFA) where required
- [ ] Established processes for access provisioning and deprovisioning
- [ ] Documented privileged access controls and audit trails
- [ ] Implemented session management and timeout policies
- [ ] Reviewed third-party access requirements and controls

**Notes/Gaps:**

---

## Section 4: Audit & Logging

- [ ] Enabled comprehensive audit logging for compliance events
- [ ] Logs capture who, what, when, where for critical actions
- [ ] Implemented log retention according to regulatory requirements
- [ ] Protected audit logs from tampering or deletion
- [ ] Configured log monitoring and alerting for compliance violations
- [ ] Documented log review procedures

**Notes/Gaps:**

## Section 5: Regulatory & Industry Standards

### Applicable Regulations
- [ ] Identified all applicable regulations and standards
- [ ] Reviewed specific requirements for each regulation
- [ ] Documented compliance controls for each requirement
- [ ] Prepared evidence collection for audits
- [ ] Validated compliance with contractual obligations

### Industry-Specific Considerations
- [ ] Healthcare (HIPAA): PHI handling, BAAs, security rule compliance
- [ ] Financial (PCI-DSS, SOX): Payment data security, financial controls
- [ ] European (GDPR): Data protection impact assessment, DPO notification
- [ ] Government (FedRAMP, FISMA): Authority to operate, continuous monitoring
- [ ] Other: [Specify]

## Section 6: Vendor & Third-Party Management

- [ ] Identified all third-party services and vendors
- [ ] Reviewed vendor security and compliance certifications
- [ ] Ensured data processing agreements (DPAs) are in place
- [ ] Validated vendor compliance with applicable regulations
- [ ] Documented vendor risk assessments
- [ ] Established vendor monitoring and review processes

## Section 7: Change Management & Configuration

- [ ] Documented change management procedures
- [ ] Required approvals obtained for production changes
- [ ] Configuration changes tracked and auditable
- [ ] Rollback procedures tested and documented
- [ ] Separation of duties enforced for critical changes

## Section 8: Incident Response & Business Continuity

- [ ] Incident response plan includes compliance breach procedures
- [ ] Breach notification procedures documented (who, when, how)
- [ ] Data breach response timelines meet regulatory requirements
- [ ] Business continuity and disaster recovery plans in place
- [ ] Regular testing of incident response procedures

**Notes/Gaps:**

## Section 9: Documentation & Training

### Compliance Documentation
- [ ] Privacy policy updated (if applicable)
- [ ] Terms of service reflect compliance requirements
- [ ] Data processing records maintained
- [ ] Compliance policies documented and accessible
- [ ] Compliance evidence organized for audits

### Training & Awareness
- [ ] Team trained on relevant compliance requirements
- [ ] Security awareness training completed
- [ ] Compliance responsibilities communicated clearly
- [ ] Regular compliance refresher training scheduled
